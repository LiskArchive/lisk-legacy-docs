Class TransferTransaction

Description

A transfer transaction sends tokens from one account to another. The specific asset properties are described in TransferAsset.

Protected applyAsset

  • applyAsset(store: StateStore): ReadonlyArray<TransactionError>

  • Overrides BaseTransaction.applyAsset

    • Defined in src/8_transfer_transaction.ts:248

    Description

    The business logic of a transaction is implemented in the applyAsset method. It applies all of the necessary changes from the received transaction to the affected account(s), by calling store.set. Calling store.get will acquire all of the relevant data. The transaction that is currently processing is the functions context, (e.g. this.amount). This transaction can be invalidated by pushing an error into the result array.

    Parameters

    • store: StateStore

      Responsible for the caching and accessing transaction and account data.

    Returns ReadonlyArray<TransactionError>

sign

  • sign(passphrase: string, secondPassphrase?: undefined | string): void

  • Inherited from VoteTransaction.sign

    • Defined in src/base_transaction.ts:594

    Description

    Method to sign the transaction.

    Example

     
    //Imports lisk-transactions
const transactions = require('@liskhq/lisk-transactions');
//Initializes a new transaction
const tx = new transactions.TransferTransaction({
   asset: {
       amount: '1',
       recipientId: '1L',
   },
   networkIdentifier: networkIdentifier,
});
//Signs the transaction with your secret passphrase
tx.sign('creek own stem final gate scrub live shallow stage host concert they');

console.log(tx.stringify());

    Parameters

    • passphrase: string

      Passphrase of the account who intends to sign the transaction.

    • Optional secondPassphrase: undefined | string

      Second passphrase of the account who intends to sign the transaction.

Protected validateAsset

  • validateAsset(): ReadonlyArray<TransactionError>

  • Overrides BaseTransaction.validateAsset

    • Defined in src/8_transfer_transaction.ts:207

    Description

    Before a transaction reaches the apply step it is validated. Check the transactions asset correctness from the schema perspective, (no access to StateStore here). Invalidate the transaction by pushing an error into the result array. Prepare the relevant information about the accounts, which will be accessible in the later steps during the apply and undo steps.
