How to enable/disable block generation

On this page, you’ll learn how to enable and disable block generation on anode for a validator.

This includes how to:

  • Enable block generation

  • Safely re-enable block generation on another node

  • Safely re-enable block generation when the validator data is lost.

  • Disable block generation

Table of Contents
  • How to enable block generation on a node
    • Get the validator data
    • Enable block generation
  • How to disable block generation on a node
  • Safely enabling block generation on another node

How to enable block generation on a node

The blockchain application needs to be running to successfully enable block generation on the node.

  1. Ensure the node is fully synchronized with the network, before enabling block generation on this node.

  2. If block generation is enabled for a validator for the first time, check out the dedicated guide How to become a validator.

  3. Ensure to adhere to the following points to avoid being punished by the network:

    1. Never use outdated validator data.

    2. Never activate block generation for the same validator on two or more nodes at the same time.

Get the validator data

Don’t trust validator data from public APIs or explorers, make sure to always use your local data!

  • Lisk Core

  • Sidechain node

./bin/run generator status --pretty

This will return the following information for a validator who imported her/his keys to the node:

  • address: The Lisk32 address of the validator.

  • enabled: true, if the delegate has block generation enabled, false if not.

  • The validator data.

Example output
{
  "info": {
    "status":
    [{
        "address": "lskqaxxmj78frvgpjgwvf4yqjjkcrr9yhn2sxxwm3",
        "height": 0,
        "maxHeightPrevoted": 0,
        "maxHeightGenerated": 0,
        "enabled": false
      }
  ]}
}
If the command returns an empty array, you probably haven’t imported your validators keys to the node. In that case, follow the guide How to become a validator to prepare the node for block generation.

Enable block generation

  • Lisk Core

  • Sidechain node

Use the generator enable command to enable block generation on your node.

Enable block generation
./bin/run generator enable lskqaxxmj78frvgpjgwvf4yqjjkcrr9yhn2sxxwm3 --height=123 --max-height-generated=101 --max-height-prevoted=101

Replace lskqaxxmj78frvgpjgwvf4yqjjkcrr9yhn2sxxwm3 with your validator address.

Verify the correctness of the values height, maxHeightPrevoted, and maxHeightPreviouslyForged by answering yes, and use your password created in step Creating the validator keys to decrypt the passphrase for block generation.

To prefill the validator data automatically, add the flag --use-status-values to the generator enable command.

lisk-core generator enable lskqaxxmj78frvgpjgwvf4yqjjkcrr9yhn2sxxwm3 --use-status-values

{"height":14814092,"maxHeightPrevoted":14814017,"maxHeightPreviouslyForged":14814025}
? Do you want to use the above values to enable block generation? yes
? Enter password to decrypt the encrypted passphrase:  **********
Updated block generation status:
{"address":"lskqaxxmj78frvgpjgwvf4yqjjkcrr9yhn2sxxwm3","enabled":true}

How to disable block generation on a node

If you wish to disable block generation on the node, follow the steps as described below.

If you would like to completely stop block generation without being punished by the network, make sure to unstake all self-stakes for a validator, before you disable block generation on the node.

  • Lisk Core

  • Sidechain node

Use the generator disable command to disable block generation on your node.

Disable block generation
./bin/run generator disable lskqaxxmj78frvgpjgwvf4yqjjkcrr9yhn2sxxwm3

Replace lskqaxxmj78frvgpjgwvf4yqjjkcrr9yhn2sxxwm3 with the address of your validator.

When prompted for a password, use the password that you defined while creating the validator keys.

Safely enabling block generation on another node

To safely enable block generation on another node, please ensure to follow the steps below:

  1. Install a new node on another server.

  2. Start the node and let it synchronize with the network. If available, it is recommended to synchronize from a snapshot, to speed up the synchronization process.

  3. Login to the server with the old node.

  4. Disable block generation on the old node.

  5. Export the hash onion seed used by the validator.

  6. Stop the old node.

  7. Export the validator info data from the old node.

    lisk-core generator export --output genInfo.json

  8. Login to the server with the new node.

  9. Restore the forger_info table.

    lisk-core generator import ./genInfo.json

  10. Import the validator keys.

  11. Import the hash onion seed used by the validator.

  12. Ensure the node is fully synchronized with the network. The height of your node should be equal to the current network height.

    lisk-core node info

  13. Get the validator data to fetch the validator info data.

  14. Enable block generation.
