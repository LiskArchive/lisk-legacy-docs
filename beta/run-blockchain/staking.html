How to stake and unstake tokens

On this page, you’ll learn how to (un-)stake tokens for a validator.

This includes how to:

  • Stake tokens

  • Un-stake tokens

  • Unlock tokens after un-stake

Introduction

Any accounts with sufficient tokens can stake them to increase the Validator weight of one or multiple validators.

Staked tokens are locked inside the account that is performing the staking. That means, the tokens used for staking stay in the account of the staker, but while they are used for staking, they are locked and cannot be used for any other purposes like for example token transfers. The staked tokens can be un-staked partially or completely again, if the account owner wishes to unlock the tokens.

Staking is particulary important for validators, because they need to stake a significant amount of their own tokens for their validator account. This is also calld self-staking.

But accounts can have incentives to stake their tokens for a different validator as well: First, it is a great way to support the most trustful and helpful validators in the community, which increases the decentralization and security of the network. Second, a lot of validators are rewarding stakers with a share of their block rewards, creating a financial incentive for users to stake their unused tokens and receive regular payout as reward.

There are various different services and tools to stake tokens, for example: Lisk Desktop, Lisk Elements, the node CLI or the Dashboard plugin.

In the following examples, the node CLI is used to create and send the respective Stake transaction.

Validator weight

Every validator has an associated validator weight. A high validator weight will put a validator in an active position to generate new blocks for the blockchain.

The minimum delegate weight required to be selected as a block generator is 1000 LSK in the Lisk Mainnet.

Formally, the validator weight is defined as the following:

validatorWeight = minimum { 10 * selfStakes , totalStakes },

  • selfStakes is the amount the validator voted for themselves, and

  • totalStakes is the sum of all staked tokens (including self-stakes) for a specific validator.

This definition implies that at least 10% of the validator weight always consist of self-stakes.

1. Start the blockchain application

If it is not already running, start the node, to conveniently create and send a transaction.

  • Lisk Core

  • Sidechain node

lisk-core start
./bin/run start

2. (Un-)Staking

Create a transaction to stake or un-stake tokens for one or multiple validators.

Up to 20 stakes and/or un-stakes can be included in a single Stake transaction.

The following parameter is required:

  • stakes: An array items with the following two properties:

    • amount: Amount of tokens to (un-)stake.

    • validatorAddress: The address of the validator to (un-)stake the tokens to.

A positive amount of tokens indicates a stake, whereas a negative amount of tokens indicates an un-stake.

2.1. Create a Stake transaction

  • Lisk Core

  • Lisk node

% ./bin/run transaction:create pos stake 100000000 --pretty --json --params='{"stakes":[{"validatorAddress":"lskqaxxmj78frvgpjgwvf4yqjjkcrr9yhn2sxxwm3","amount":15000000000},{"validatorAddress":"lske5sqed53fdcs4m9et28f2k7u9fk6hno9bauday","amount":-5000000000}]}'

  • Lisk Core

  • Sidechain node

3. Unlocking tokens after un-stake

In case you un-staked tokens from one or multiple validators and wish to be able to freely use the tokens for other purposes than staking, it is required to unlock the tokens.

To do this, send a corresponding Unlock transaction to the node.

3.1. Create the Unlock transaction

The Unlock transaction will unlock all tokens which are pending, i.e. currently locked in the account, without being used for staking.

  • Lisk Core

  • Lisk node

% ./bin/run transaction:create pos unlock 100000000 --pretty --json

Next, you will be prompted for the passphrase of your account.

? Please enter passphrase:  [hidden]
? Please re-enter passphrase:  [hidden]
